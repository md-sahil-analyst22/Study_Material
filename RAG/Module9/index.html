What is Retrieval-Augmented Generation (RAG)?

Core Concept

Retrieval-Augmented Generation (RAG) is an AI framework that enhances large language models (LLMs) by retrieving relevant information from external knowledge sources before generating responses. This approach addresses key limitations of LLMs:

RAG Advantages
  • Reduces hallucinations by grounding responses in retrieved facts
  • Provides access to up-to-date information beyond training cut-off
  • Enables domain-specific applications without retraining
  • Improves transparency with source attribution
  • Reduces computational costs compared to fine-tuning
RAG Components
  1. Retriever: Searches knowledge base for relevant information
  2. Vector Store: Stores document embeddings for efficient search
  3. Generator: LLM that produces final answer using retrieved context
  4. Embedding Model: Converts text to numerical vectors

Vector Search Engines: The Core Technology

Why Vector Search?

Traditional keyword-based search has limitations in understanding semantic meaning. Vector search transforms text into high-dimensional vectors (embeddings) where semantic similarity is represented by spatial proximity.

Speed

Fast approximate nearest neighbor search even in billion-scale datasets

Semantic Understanding

Captures meaning beyond keywords (synonyms, context, concepts)

Scalability

Efficiently handles high-dimensional data (100+ dimensions)

Key Vector Search Algorithms

Algorithm Type Best For Time Complexity Accuracy
Flat Index (Brute Force) Exact Search Small datasets, maximum accuracy O(n) 100%
IVF (Inverted File) Approximate Medium datasets, balanced speed/accuracy O(√n) 90-95%
HNSW (Hierarchical NSW) Approximate Large datasets, high speed & accuracy O(log n) 90-99%
LSH (Locality-Sensitive Hashing) Approximate High-dimensional sparse data O(1) average 80-90%

HNSW Algorithm Deep Dive

Hierarchical Navigable Small World Graphs

HNSW is a state-of-the-art approximate nearest neighbor search algorithm that organizes data in multiple layers, similar to a skyscraper with express elevators (top layers) and local corridors (bottom layers).

HNSW Multi-Layer Structure

How HNSW Search Works: Step-by-Step
  1. Start at Top Layer: Begin search at a random entry point in the highest layer (fewest points, longest connections)
  2. Greedy Traversal: Move to neighbor closest to target query in current layer
  3. Layer Descent: When no closer neighbors exist, move down one layer
  4. Repeat: Continue greedy search and layer descent until reaching bottom layer
  5. Final Search: Perform final greedy search in bottom layer (all points)
  6. Return Results: Return approximate nearest neighbors with 90-99% recall
Key HNSW Parameters
M (Max Connections)

Controls how many neighbors each node connects to. Higher M = better accuracy but more memory.

Default: 16 Range: 4-64
efConstruction

Search breadth during index construction. Higher values = better graph quality but slower build.

Default: 200 Range: 100-2000
efSearch

Search breadth during querying. Higher values = better accuracy but slower search.

Default: 16 Range: 16-512
mL (Level Multiplier)

Controls probability distribution for layer assignment. Affects hierarchy shape.

Default: 1/log(M)
HNSW Limitations: Best for mostly static datasets. Frequent updates can degrade performance, requiring periodic index rebuilding.

FAISS Deep Dive: Facebook AI Similarity Search

What is FAISS?

FAISS (Facebook AI Similarity Search) is a library developed by Meta for efficient similarity search and clustering of dense vectors. It's optimized for use on a single machine with CPU or GPU acceleration.

FAISS Architecture Components

FAISS Index Types Explained
Index Type Mechanism Best For Pros Cons
IndexFlatL2 Brute-force L2 distance comparison Small datasets (< 10K vectors), maximum accuracy 100% accurate, simple Slow for large datasets
IndexIVFFlat Inverted file with Voronoi cells Medium datasets, balanced approach Fast, memory efficient Requires training, may miss neighbors
IndexHNSW Hierarchical navigable small world graph Large datasets, high performance Very fast, high accuracy Higher memory usage
IndexLSH Locality-sensitive hashing High-dimensional sparse data Memory efficient, fast Lower accuracy
Basic FAISS Implementation Example

import numpy as np
import faiss

# Generate sample vectors (1000 vectors of dimension 128)
dimension = 128
num_vectors = 1000
vectors = np.random.random((num_vectors, dimension)).astype('float32')

# Create a Flat index (brute-force, exact search)
index_flat = faiss.IndexFlatL2(dimension)
index_flat.add(vectors)
print(f"Flat index: {index_flat.ntotal} vectors")

# Create an IVF index for faster approximate search
nlist = 100  # number of Voronoi cells
quantizer = faiss.IndexFlatL2(dimension)
index_ivf = faiss.IndexIVFFlat(quantizer, dimension, nlist)
index_ivf.train(vectors)  # IVF requires training
index_ivf.add(vectors)
index_ivf.nprobe = 10  # search in 10 nearest cells
print(f"IVF index: {index_ivf.ntotal} vectors")

# Create an HNSW index (high performance)
M = 16  # connections per node
index_hnsw = faiss.IndexHNSWFlat(dimension, M)
index_hnsw.add(vectors)
print(f"HNSW index: {index_hnsw.ntotal} vectors")

# Perform a search query
query_vector = np.random.random((1, dimension)).astype('float32')
k = 5  # number of nearest neighbors to retrieve

# Search with Flat index (exact)
distances_flat, indices_flat = index_flat.search(query_vector, k)
print(f"Flat search results: {indices_flat[0]}")

# Search with IVF index (approximate)
distances_ivf, indices_ivf = index_ivf.search(query_vector, k)
print(f"IVF search results: {indices_ivf[0]}")

# Search with HNSW index (approximate)
distances_hnsw, indices_hnsw = index_hnsw.search(query_vector, k)
print(f"HNSW search results: {indices_hnsw[0]}")
Extending FAISS with Milvus

Chroma DB: AI-Native Vector Database

What is Chroma DB?

Chroma DB is an open-source embedding database designed specifically for AI applications. It simplifies building LLM apps by providing a complete solution for storing and querying embeddings with metadata.

Key Features
  • Built-in embedding functions and vector store
  • Native metadata filtering and querying
  • Simple API with LangChain integration
  • Multiple distance metrics (L2, cosine, IP)
  • HNSW as the default indexing algorithm
  • Support for both in-memory and persistent storage
  • Production-ready with client/server mode
Ideal Use Cases
  • Rapid prototyping of AI applications
  • Semantic search implementations
  • Question-answering systems with source retrieval
  • Recommendation systems with metadata filtering
  • Document similarity and clustering
Chroma DB Implementation Example

import chromadb
from chromadb.config import Settings

# Initialize Chroma client
chroma_client = chromadb.Client(Settings(
    chroma_db_impl="duckdb+parquet",
    persist_directory="./chroma_db"  # Optional: persist to disk
))

# Create a collection (similar to a table)
collection = chroma_client.create_collection(
    name="my_documents",
    metadata={"hnsw:space": "cosine"}  # Use cosine similarity
)

# Add documents with embeddings and metadata
documents = [
    "The quick brown fox jumps over the lazy dog",
    "Machine learning is a subset of artificial intelligence",
    "Vector databases enable semantic search capabilities",
    "Python is a popular programming language for AI"
]

metadata = [
    {"source": "example1", "category": "animals"},
    {"source": "example2", "category": "AI"},
    {"source": "example3", "category": "databases"},
    {"source": "example4", "category": "programming"}
]

ids = ["doc1", "doc2", "doc3", "doc4"]

# Add to collection (Chroma automatically generates embeddings)
collection.add(
    documents=documents,
    metadatas=metadata,
    ids=ids
)

# Query with metadata filtering
results = collection.query(
    query_texts=["What is machine learning?"],
    n_results=2,
    where={"category": "AI"}  # Metadata filter
)

print("Query results:", results['documents'][0])

# Get all items with metadata filtering
all_ai_docs = collection.get(
    where={"category": "AI"}
)
print("AI documents:", all_ai_docs['documents'])

# Update a document
collection.update(
    ids=["doc2"],
    documents=["Machine learning and deep learning are AI subsets"],
    metadatas=[{"source": "example2", "category": "AI", "updated": True}]
)

# Delete documents
collection.delete(ids=["doc1"])
Chroma Limitation: Chroma DB only supports HNSW indexing (no IVF, LSH, or other FAISS index types). This simplifies usage but reduces flexibility for specialized use cases.

Implementation Examples

Semantic Search with FAISS: Complete Example

This example demonstrates building a semantic search engine using FAISS with the 20 Newsgroups dataset.


import numpy as np
import faiss
import re
from sklearn.datasets import fetch_20newsgroups
import tensorflow_hub as hub

# Load the Universal Sentence Encoder
embed = hub.load("https://tfhub.dev/google/universal-sentence-encoder/4")

# Function to generate embeddings
def embed_text(text):
    return embed(text).numpy()

# Preprocessing function
def preprocess_text(text):
    # Remove email headers
    text = re.sub(r'^From:.*\\n?', '', text, flags=re.MULTILINE)
    # Remove email addresses
    text = re.sub(r'\\S*@\\S*\\s?', '', text)
    # Remove punctuations and numbers
    text = re.sub(r'[^a-zA-Z\\s]', '', text)
    # Convert to lowercase
    text = text.lower()
    # Remove excess whitespace
    text = re.sub(r'\\s+', ' ', text).strip()
    return text

# Load and preprocess dataset
newsgroups = fetch_20newsgroups(subset='all')
documents = newsgroups.data
processed_documents = [preprocess_text(doc) for doc in documents]

# Generate embeddings for documents (in batches for large datasets)
batch_size = 100
embeddings_list = []

for i in range(0, len(processed_documents), batch_size):
    batch = processed_documents[i:i+batch_size]
    batch_embeddings = embed_text(batch)
    embeddings_list.append(batch_embeddings)
    
X_use = np.vstack(embeddings_list)

# Create FAISS index
dimension = X_use.shape[1]
index = faiss.IndexHNSWFlat(dimension, 16)  # M=16 connections
index.add(X_use.astype('float32'))

# Search function
def search(query_text, k=5):
    preprocessed_query = preprocess_text(query_text)
    query_vector = embed_text([preprocessed_query])
    distances, indices = index.search(query_vector.astype('float32'), k)
    return distances, indices

# Example query
query_text = "machine learning algorithms"
distances, indices = search(query_text)

print("Search results for query:", query_text)
for i, idx in enumerate(indices[0]):
    print(f"Result {i+1} (Distance: {distances[0][i]:.4f}):")
    print(processed_documents[idx][:200] + "...")
    print()
YouTube Summarizer with RAG and FAISS

This example shows a complete RAG application that summarizes YouTube videos and answers questions about their content.


# Key components from the YouTube summarizer project

def get_youtube_transcript(video_url):
    """Extract transcript from YouTube video"""
    from youtube_transcript_api import YouTubeTranscriptApi
    import re
    
    # Extract video ID from URL
    video_id = re.search(r'v=([a-zA-Z0-9_-]{11})', video_url)
    if video_id:
        video_id = video_id.group(1)
        transcript = YouTubeTranscriptApi.get_transcript(video_id)
        return " ".join([t['text'] for t in transcript])
    return None

def chunk_text(text, chunk_size=500, overlap=50):
    """Split text into overlapping chunks"""
    from langchain.text_splitter import RecursiveCharacterTextSplitter
    
    text_splitter = RecursiveCharacterTextSplitter(
        chunk_size=chunk_size,
        chunk_overlap=overlap,
        length_function=len,
        separators=["\\n\\n", "\\n", " ", ""]
    )
    return text_splitter.split_text(text)

def create_vector_store(chunks, embedding_model):
    """Create FAISS vector store from text chunks"""
    from langchain.vectorstores import FAISS
    
    vector_store = FAISS.from_texts(chunks, embedding_model)
    return vector_store

def answer_question(query, vector_store, llm):
    """Retrieve relevant context and generate answer"""
    from langchain.chains import RetrievalQA
    from langchain.prompts import PromptTemplate
    
    # Create prompt template
    prompt_template = """Use the following context to answer the question.
    If you don't know the answer, say you don't know.
    
    Context: {context}
    
    Question: {question}
    
    Answer: """
    
    PROMPT = PromptTemplate(
        template=prompt_template, 
        input_variables=["context", "question"]
    )
    
    # Create retrieval QA chain
    qa_chain = RetrievalQA.from_chain_type(
        llm=llm,
        chain_type="stuff",
        retriever=vector_store.as_retriever(search_kwargs={"k": 3}),
        chain_type_kwargs={"prompt": PROMPT}
    )
    
    return qa_chain.run(query)

# Main workflow for YouTube Q&A
def youtube_qa_system(video_url, question):
    """
    Complete workflow:
    1. Extract YouTube transcript
    2. Chunk the transcript
    3. Create vector store (FAISS)
    4. Retrieve relevant context and generate answer
    """
    # Step 1: Get transcript
    transcript = get_youtube_transcript(video_url)
    if not transcript:
        return "Could not retrieve transcript"
    
    # Step 2: Chunk text
    chunks = chunk_text(transcript)
    
    # Step 3: Create embeddings and vector store
    # (Assuming embedding_model and llm are initialized)
    vector_store = create_vector_store(chunks, embedding_model)
    
    # Step 4: Answer question
    answer = answer_question(question, vector_store, llm)
    return answer
Production Tip: For production systems, add caching (Redis), async processing, monitoring (Prometheus), and load balancing. Consider using dedicated vector databases like Pinecone or Weaviate for large-scale deployments.
