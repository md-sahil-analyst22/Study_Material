üéØWhat is Gradio?

Gradio is an open-source Python library that enables developers to quickly build and share interactive web interfaces for machine learning models, APIs, or any Python function. No JavaScript, CSS, or web hosting experience required!

‚öôÔ∏èHow Gradio Works

1. Write Python

Define your function with application logic

2. Create Interface

Use Gradio's Interface class to specify inputs/outputs

3. Launch Server

4. Access & Share

Use via local or public URL

üîßThe Interface Class

The gr.Interface class is the core of Gradio. It wraps any Python function with a user interface.

Parameter Description Example
fn The Python function to wrap with UI predict, classify, generate_text
inputs Gradio component(s) for input. Must match function parameters gr.Textbox(), gr.Image(), gr.Slider()
outputs Gradio component(s) for output. Must match return values gr.Label(), gr.Textbox(), gr.Image()

üí° Simple Example

import gradio as gr def greet(name, intensity): return "Hello, " + name + "!" * int(intensity) demo = gr.Interface( fn=greet, inputs=["text", "slider"], outputs=["text"] ) demo.launch(server_name="127.0.0.1", server_port=7860)

üì•Common Input Types

Component Description Use Case
gr.Textbox() Expandable text input field Text generation, Q&A bots, text analysis
gr.Image() Image upload/select component Image classification, object detection
gr.Slider() Slider for numeric range selection Parameter tuning, intensity control
gr.Checkbox() Boolean True/False selection Feature toggles, binary options
gr.Dropdown() Dropdown selection menu Category selection, model choice
gr.Radio() Radio button selection (one choice) Exclusive options, single selection
gr.CheckboxGroup() Multiple checkbox selections Multi-label classification
gr.File() File upload component Document processing, CSV analysis

üì§Common Output Types

gr.Textbox()

  • Displays text output
  • Perfect for generated text, responses
  • Can be expandable

gr.Label()

  • Displays classification results
  • Shows confidence probabilities
  • Use num_top_classes to limit results

gr.Image()

  • Displays generated images
  • Image manipulation results
  • Supports various formats

gr.Number()

  • Displays numeric output
  • Calculation results
  • Metrics and scores

Architecture Flow

1. Model Loading

Load pretrained ResNet-18 from PyTorch Hub

import torch model = torch.hub.load('pytorch/vision:v0.6.0', 'resnet18', pretrained=True).eval()
‚Üì

2. Prediction Function

Process image and return predictions

def predict(inp): inp = transforms.ToTensor()(inp).unsqueeze(0) with torch.no_grad(): prediction = torch.nn.functional.softmax(model(inp)[0], dim=0) confidences = {labels[i]: float(prediction[i]) for i in range(1000)} return confidences
‚Üì

3. Gradio Interface

Create user interface with inputs and outputs

gr.Interface( fn=predict, inputs=gr.Image(type="pil"), outputs=gr.Label(num_top_classes=3), examples=["/content/lion.jpg", "/content/cheetah.jpg"] ).launch()

üí° Key Concepts

  • Softmax Function: Converts raw model outputs (logits) into probabilities that sum to 1
  • PIL Image: Python Imaging Library format for image handling
  • Examples Parameter: Pre-populate interface with sample inputs
  • num_top_classes: Limit displayed classifications to top N classes

ü§ñBuilding a RAG QA Bot

RAG (Retrieval-Augmented Generation) combines document retrieval with LLM generation to answer questions based on specific documents.

Complete RAG Pipeline

üìÑ Document Loading

Load PDF documents using PyPDFLoader

from langchain_community.document_loaders import PyPDFLoader loader = PyPDFLoader(file.name) documents = loader.load()
‚Üì

‚úÇÔ∏è Text Splitting

Break documents into manageable chunks

from langchain.text_splitter import RecursiveCharacterTextSplitter splitter = RecursiveCharacterTextSplitter( chunk_size=1000, chunk_overlap=200 ) chunks = splitter.split_documents(documents)
‚Üì

üßÆ Embedding Generation

Convert text chunks to vector embeddings

from langchain_ibm import WatsonxEmbeddings embeddings = WatsonxEmbeddings( model_id="ibm/slate-125m-english-rtrvr", url="https://us-south.ml.cloud.ibm.com", project_id="skills-network" )
‚Üì

üíæ Vector Storage

Store embeddings in ChromaDB vector database

from langchain_community.vectorstores import Chroma vectordb = Chroma.from_documents(chunks, embeddings)
‚Üì

üîç Retrieval

Search for relevant document chunks

retriever = vectordb.as_retriever()
‚Üì

üéØ QA Chain

Combine retriever with LLM for question answering

from langchain.chains import RetrievalQA qa = RetrievalQA.from_chain_type( llm=watsonx_llm, chain_type="stuff", retriever=retriever, return_source_documents=True )

Example 4: Sentiment Analysis with Multiple Inputs

Analyze text sentiment with category selection:

import gradio as gr def analyze_sentiment(text, categories): # Simple keyword-based sentiment (in practice, use ML model) positive_words = [ 'good', 'great', 'excellent', 'amazing', 'wonderful' ] negative_words = [ 'bad', 'poor', 'terrible', 'awful', 'horrible' ] text_lower = text.lower() pos_count = sum( 1 for word in positive_words if word in text_lower ) neg_count = sum( 1 for word in negative_words if word in text_lower ) if pos_count > neg_count: sentiment = "Positive" score = min(pos_count * 0.3, 1.0) elif neg_count > pos_count: sentiment = "Negative" score = min(neg_count * 0.3, 1.0) else: sentiment = "Neutral" score = 0.5 category_info = f"Analyzing as: {', '.join(categories)}" return { "Positive": score if sentiment == "Positive" else 0.0, "Negative": score if sentiment == "Negative" else 0.0, "Neutral": score if sentiment == "Neutral" else 0.0 }, category_info demo = gr.Interface( fn=analyze_sentiment, inputs=[ gr.Textbox( lines=5, placeholder="Enter text to analyze...", label="Text" ), gr.CheckboxGroup( ["Product Review", "Customer Feedback", "Social Media"], label="Analysis Categories" ) ], outputs=[ gr.Label(label="Sentiment Scores"), gr.Textbox(label="Category Info") ], title="Sentiment Analyzer", examples=[ ["This product is amazing! I love it!", ["Product Review"]], ["Terrible service, very disappointed.", ["Customer Feedback"]] ] ) demo.launch()

Example 7: Image Classification with Examples

Complete image classification with pre-loaded examples:

import gradio as gr import torch from torchvision import transforms from PIL import Image import requests # Load pretrained model model = torch.hub.load('pytorch/vision:v0.6.0', 'resnet18', pretrained=True).eval() # Download labels response = requests.get("https://git.io/JJkYN") labels = response.text.split("\n") def classify_image(image): # Preprocess image transform = transforms.Compose([ transforms.Resize(256), transforms.CenterCrop(224), transforms.ToTensor(), transforms.Normalize( mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225] ) ]) img_tensor = transform(image).unsqueeze(0) # Make prediction with torch.no_grad(): outputs = model(img_tensor) probabilities = torch.nn.functional.softmax(outputs[0], dim=0) # Get top 5 predictions top5_prob, top5_indices = torch.topk(probabilities, 5) results = {} for i in range(5): results[labels[top5_indices[i]]] = float(top5_prob[i]) return results # Create interface demo = gr.Interface( fn=classify_image, inputs=gr.Image(type="pil", label="Upload or drag an image"), outputs=gr.Label(num_top_classes=5, label="Top 5 Predictions"), title="üñºÔ∏è Image Classification with ResNet-18", description="Upload an image to classify it into one of 1000 ImageNet categories.", article="Model: ResNet-18 pretrained on ImageNet dataset.", examples=[ ["path/to/example1.jpg"], ["path/to/example2.jpg"] ], allow_flagging="never" ) demo.launch()

Example 8: Text Generation with Temperature Control

Build a text generator with adjustable parameters:

import gradio as gr from langchain_ibm import WatsonxLLM # Initialize LLM def initialize_llm(temperature, max_tokens): parameters = { "max_new_tokens": max_tokens, "temperature": temperature } llm = WatsonxLLM( model_id="ibm/granite-3-3-8b-instruct", url="https://us-south.ml.cloud.ibm.com", project_id="skills-network", params=parameters ) return llm def generate_text(prompt, temperature, max_tokens): try: llm = initialize_llm(temperature, max_tokens) response = llm.invoke(prompt) return response except Exception as e: return f"Error: {str(e)}" demo = gr.Interface( fn=generate_text, inputs=[ gr.Textbox( lines=5, placeholder="Enter your prompt here...", label="Prompt" ), gr.Slider( minimum=0.1, maximum=1.0, value=0.7, step=0.1, label="Temperature (creativity)" ), gr.Slider( minimum=50, maximum=500, value=150, step=50, label="Max Tokens" ) ], outputs=gr.Textbox( label="Generated Text", lines=10 ), title="ü§ñ AI Text Generator", description="Generate text using IBM Granite model with adjustable parameters.", examples=[ [ "Write a short story about a robot learning to paint", 0.8, 200 ], [ "Explain quantum computing in simple terms", 0.5, 150 ] ] ) demo.launch()
